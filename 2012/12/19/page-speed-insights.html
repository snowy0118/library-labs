大学図書館サイトのページスピード・スコアを調べました

大学図書館サイトのページスピードをGoogleのPageSpeed InsightsのAPIを使って調べました。

今回調べたのは以下の都道府県です。この一年間でこのサイトへのアクセスが多いところを選びました。図書館のリストはカーリルのAPIを使って出力しました。サイトのURLはカーリルに登録されているURLです。URLが変更になっている図書館は404のページやリダイレクトページのスコアになっている場合があります。

スコア以外の分析結果はPageSpeed Insightsで確認してください。

  • 東京都
  • 京都府
  • 神奈川県
  • 大阪府
  • 愛知県
  • 北海道
  • 福岡県

PageSpeed Insightsはウェブページのコンテンツを分析してスコアを出します。主なチェック項目は次のとおりです。これらを実践することでウェブページは高速化し、ユーザーエクスペリエンスの向上に繋がります。

  • CSS @import を使用しない
  • CSS をドキュメント ヘッドに含める
  • CSS スプライトに画像をまとめる
  • JavaScript の解析を遅延する
  • JavaScript を縮小する
  • キャッシュ バリデータを指定する
  • スタイルシートとスクリプトの順序を最適化する
  • リクエスト サイズを減らす
  • リクエストの直列化を減らす
  • リダイレクトの回数を減らす
  • 不要なリフローをなくす
  • 変更後のサイズで画像を提供する
  • 小さな CSS をインライン化する
  • 小さな JavaScript をインライン化する
  • 文字セットを指定する
  • 画像のサイズを指定する
  • 誤りのあるリクエストを送信しない
  • 重複するリソースは 1 つの URL から提供する
  • 長時間実行されるスクリプトは避ける
  • 静的リソースからクエリ文字列を削除する
  • 非同期リソースを使用する
  • 圧縮を有効にする
  • 画像を最適化する
  • CSS を縮小する
  • HTML を縮小する
